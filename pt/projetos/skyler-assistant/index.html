Skyler Assistant

Skyler é o assistente de portfólio público de Patrick Araujo, focado no contexto do projeto, conjunto técnico, experiência profissional e orientação de contato.

O assistente foi projetado para orquestração previsível: saudação simples, resposta local, uso manual RAG e chamadas LLM somente quando necessário.

Tempo de execução primárioCloudflare Worker (TypeScript)
Camada de dadosD1 + KV (`DB`, `CACHE`, `SESSIONS`)
RAG Fontes`manual-rag.ts` + páginas pklavc.com em cache
InterfaceWidget flutuante global + página de assistente dedicada
VozWeb Speech API com preferência de voz feminina
ConfiabilidadeFallback do provedor e cache de resposta

Arquitetura do sistema

Skyler separa as responsabilidades claramente. O frontend lida com interação e voz. O Worker orquestra memória, reconhecimento de intenção RAG e chamadas de provedor. O modelo lida com respostas diretas para solicitações fora do tópico e respostas guiadas pelo portfólio quando a intenção do portfólio é detectada.

Usuário (IU da Web) -> /chat API -> Orquestrador de Trabalhadores
Verificação de saudação -> Intenção do portfólio + RAG Portão -> Construtor de prompts
Provedor Groq ou Alternativa do OpenRouter -> Resposta + Armazenamento

Fluxo de decisão em tempo de execução

A entrada chega -> Saudação simples? sim Resposta rápida de saudação local
Se não for saudação -> Pergunta do portfólio? sim Mesclar manual RAG + contexto do site em cache
Se não for uma solicitação de portfólio -> Ligue para LLM diretamente falhar? Usar substituto de provedor

Experiência de front-end

  • Iniciador de ícones flutuantes nas páginas do site usando /images/icon_skyler.webp
  • Assistente interativo dedicado em /skyler-assistente/
  • Enter envia mensagem, Shift+Enter adiciona quebra de linha
  • Saída de streaming opcional e alternância de saída de voz
  • Uso público/convidado sem login obrigatório

Detalhes do trabalhador e dos dados

  • Entrada: src/chatbot/worker/src/index.ts
  • RAG manual: src/chatbot/worker/src/manual-rag.ts
  • Cache KV: cache de resposta, cache de incorporação, cache de contexto do site
  • D1: usuários, conversas, mensagens, eventos analíticos, versões de prompt
  • Ligações: DB, CACHE, SESSIONS

RAG Cobertura

O manual RAG inclui identidade, foco de função, pilha, links, empresas, métricas de impacto e contexto de credenciais atual. Ele é mesclado com páginas de sites públicos em cache somente quando o usuário está perguntando sobre tópicos do portfólio, o que mantém diretas as respostas fora do tópico e reduz fundamentações indesejadas.

  • Identidade e posicionamento (backend, integrações, arquitetura)
  • Orientação para descoberta de projetos e blogs
  • Referências de contato e perfil (GitHub, LinkedIn, email)
  • Credenciais, empresas e projetos conhecidos do contexto interno e de páginas públicas
  • Regras de resposta em texto simples para respostas de cursos, empresas e projetos

Estratégia de confiabilidade e fallback

  • Caminho de execução do provedor principal com novas tentativas
  • Provedor substituto automático quando o primário falha
  • Caminho de saudação local mais reconhecimento de intenção RAG para reduzir chamadas de modelo desnecessárias
  • Cache de resposta para minimizar gastos repetidos de token
